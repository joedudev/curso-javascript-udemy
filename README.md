## ⚙️ Commit Strategy & Versioning

This repository follows the **Conventional Commits** specification to maintain a clean and readable history. Every commit is structured as:  
`<type>(<scope>): <emoji> <short description>`

### 🛠 Standard Types Used:

- **`feat`**: New lesson or lab exercise (e.g., `feat(blackjack): 🃏 add deck logic`).
- **`fix`**: Correcting a bug in an exercise.
- **`docs`**: Documentation changes (like this README).
- **`refactor`**: Code optimization without changing functionality.
- **`chore`**: Maintenance (Vite setup, dependencies, etc.).

### 🏷️ Tagging System:

I use **Semantic Versioning** tags to mark the completion of each module.

- `vX.0.0` for Major Labs/Projects.
- `v0.X.0` for Conceptual Modules.
